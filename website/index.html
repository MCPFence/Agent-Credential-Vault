Core Problem

3 Fatal Flaws in How Agents Use Credentials

Without a butler, every credential you hand to an agent is a liability

🔓

Credentials in Plain Sight

API keys pasted into prompts. Tokens stored in .env files. The agent's entire context window can see your secrets — including logs, debug output, and injected prompts.

$ export OPENAI_API_KEY=sk-proj-abc123...
$ python agent.py  # Agent can print(os.environ["OPENAI_API_KEY"])
⚠

Permissions Out of Control

You gave your agent a GitHub token just to read code. But that token has full repo access — delete branches, modify workflows, push malicious code. Nothing stops it.

scope: repo (full control)  ← you only wanted git clone
👁

No Tracking & No Revocation

When did your agent use your Gmail token? What emails did it read? You have no idea. Worse: the agent keeps the password forever — task done, token still alive.

Task finished 30 min ago... Token still valid... No audit log.
Core Solution

Use Without Seeing

The butler handles all credential operations. The agent never touches the plaintext.

Before — No Butler
# Agent gets the plaintext password
token = vault.get("gmail_oauth")
print(token)
# → ya29.a0AfB_byC8x...
# Agent sees it! Logs see it!
# Prompt injection sees it too!
After — Credential Butler
# Agent runs the command, can't see the password
ais exec --cred vcred_xxx \
  --env TOKEN=access_token \
  -- curl -H "Bearer $TOKEN" api.gmail.com
# → Returns email data normally
# Agent never sees the token plaintext

Agents use credentials to get work done, but never touch the plaintext. Auto-revoked when done. Safe from prompt injection.

Product Highlights

6 Promises from Your Butler

Every design choice is about keeping you in control in the age of AI agents

🔒

Use Without Seeing

ais exec injects credentials into a subprocess's environment. The agent (parent process) cannot read them. Output is scrubbed in real time — any leaked secret is replaced with [AIS:REDACTED].

ais exec --cred <id> -- command
⏰

Auto-Revoke on Finish

When the task ends, all credential access is instantly revoked. No "agent still holds the password" problem. Credential lifetime = task lifetime.

task finish → all tokens revoked
📑

Full Audit Trail

Every credential use is logged: which agent, which credential, when, and why. A complete audit chain so you always know what happened with your accounts.

who + what + when + why
🌐

One Tool, Every Scenario

Whether it's curl for HTTP, Python initializing an SDK, Node.js reading Notion, or kubectl managing a cluster — ais exec covers them all with a single command.

HTTP / Python / Node / Go / Shell
📱

Device + App Binding

Each agent identity is bound to one device and one app. Identity files can't be copied to another machine. Lost a device? Revoke that identity — others remain unaffected.

1 device + 1 app = 1 identity
🛠

Desktop App + CLI

Desktop app to manage credentials, approve agents, and view audit logs visually. Go CLI for agents to securely use credentials from Claude Code, Cursor, or any terminal.

Desktop App + ais CLI

Usage Examples

# Python SDK — agent can't see the API key
ais exec --cred vcred_openai --env OPENAI_API_KEY=api_key -- python my_agent.py

# Notion cookie injection — agent can't see the cookie
ais exec --cred vcred_notion --env COOKIES=cookies -- node notion_reader.js

# Multiple credentials at once
ais exec --cred vcred_gh --env GH=api_key \\
         --cred vcred_slack --env SLACK=access_token \\
         -- python multi_service.py

# Credential as temp file (e.g. kubeconfig) — auto-deleted after exit
ais exec --cred vcred_k8s --file /tmp/.kube/config=kubeconfig_data -- kubectl get pods
