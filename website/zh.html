Agent 凭证管家

你的 Agent 需要账号密码才能干活，但你不想把密码直接给它。
凭证管家帮你管 —— Agent 能用，但看不见、带不走、用完自动收回。

快速上手 了解我们解决的问题
Core Problem

Agent 使用凭证的 3 个致命问题

没有管家管控，你的每个账号都在"裸奔"

🔓

凭证裸奔

API Key 直接粘贴到 prompt 里，Token 明文写在 .env 文件中。 Agent 的整个上下文窗口都能看到你的密码 —— 包括日志、调试输出、甚至被注入的恶意 prompt。

$ export OPENAI_API_KEY=sk-proj-abc123...
$ python agent.py  # Agent 可以 print(os.environ["OPENAI_API_KEY"])
⚠

权限失控

你给了 Agent 一个 GitHub Token，本意只让它读代码。 但这个 Token 有 repo 全部权限 —— 删分支、改 workflow、推送恶意代码，Agent 都能做。 没有任何机制约束它只做你要求的事。

scope: repo (full control)  ← 你只想让它 git clone
👁

无法追踪 & 无法收回

Agent 什么时候用了你的 Gmail Token？读了哪些邮件？发了什么消息？ 你完全不知道。更糟糕的是，Agent 拿到密码后永远持有 —— 任务结束了，密码还在它手里。

Task finished 30 min ago... Token still valid... No audit log.
Core Solution

凭证管家的核心能力 — 可用不可见

管家替 Agent 操办一切凭证事务，Agent 全程接触不到明文。这不是妥协，是管家的本分。

Before — 没有管家
# Agent 直接拿到明文密码
token = vault.get("gmail_oauth")
print(token)
# → ya29.a0AfB_byC8x...
# Agent 看到了！日志看到了！
# Prompt injection 也看到了！
After — 凭证管家
# Agent 只执行命令，看不到密码
ais exec --cred vcred_xxx \
  --env TOKEN=access_token \
  -- curl -H "Bearer $TOKEN" api.gmail.com
# → 正常返回邮件数据
# Agent 全程看不到 Token 明文

Agent 能用凭证办事，但全程接触不到明文。用完自动收回，Prompt Injection 也拿不走。

Product Highlights

管家的 6 项承诺

每一个设计决策，都是为了让你在 AI Agent 时代保持掌控

🔒

可用不可见

ais exec 将凭证注入到子进程的环境变量中，Agent（父进程）无法读取。 子进程的输出经过实时清洗，任何泄漏的明文都被替换为 [AIS:REDACTED]。

ais exec --cred <id> -- command
⏰

用完即释放

任务结束调用 ais task finish，所有凭证访问权立即失效。 不存在"Agent 永久持有密码"的问题。凭证的生命周期 = 任务的生命周期。

task finish → all tokens revoked
📑

全程审计

每次凭证使用都记录：哪个 Agent、用了哪个凭证、什么时间、为什么目的。 完整的审计链让你随时回溯，清楚知道你的账号发生了什么。

who + what + when + why
🌐

一个工具，全场景覆盖

不管是 curl 调 HTTP API、Python 脚本初始化 SDK、Node.js 读 Notion、还是 kubectl 管集群 —— ais exec 一个命令搞定所有场景。

HTTP / Python / Node / Go / Shell
📱

设备 + 应用绑定

每个 Agent 身份绑定到一台设备 + 一个应用。身份文件无法拷贝到其他机器使用。 丢了一台设备？撤销该设备的身份，不影响其他设备。

1 device + 1 app = 1 identity
🛠

桌面应用 + CLI 双通道

桌面应用可视化管理凭证、审批 Agent、查看审计日志 —— 点点鼠标就能掌控一切。 Go CLI 让 Agent 在 Claude Code、Cursor、终端中通过命令行安全使用凭证。

Desktop App + ais CLI

更多使用场景

# Python SDK 初始化 — Agent 看不到 API Key
ais exec --cred vcred_openai --env OPENAI_API_KEY=api_key -- python my_agent.py

# Notion Cookie 注入 — Agent 看不到 Cookie
ais exec --cred vcred_notion --env COOKIES=cookies -- node notion_reader.js

# 多凭证同时使用
ais exec --cred vcred_gh --env GH=api_key \\
         --cred vcred_slack --env SLACK=access_token \\
         -- python multi_service.py

# 凭证写入临时文件（如 kubeconfig）— 命令结束后自动删除
ais exec --cred vcred_k8s --file /tmp/.kube/config=kubeconfig_data -- kubectl get pods
